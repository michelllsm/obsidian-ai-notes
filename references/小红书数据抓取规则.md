# 小红书数据抓取规则

## 测试结论（2026-04-10）

**核心发现**：小红书有**强登录态依赖**，裸链接无法直接抓取有效内容。

### 测试结果对比
| 方式              | 链接类型            | 结果      | 数据完整性        | 成功率  |
| --------------- | --------------- | ------- | ------------ | ---- |
| curl SSR        | 裸链接             | ❌ 失败    | 空数据/几十KB     | <10% |
| curl SSR        | 带xsec_token完整链接 | ✅ 成功    | 完整（正文+图片+互动） | >90% |
| web_fetch       | 裸链接             | ⚠️ 部分成功 | 仅公开文字，无图片/互动 | ~30% |
| web_fetch       | 带xsec_token链接   | ✅ 成功    | 完整           | >90% |
| xiaohongshu MCP | 任意链接            | ✅ 成功    | 最完整（含评论分页）   | >95% |

### AI执行决策树

```
小红书抓取需求
├── 是否有xiaohongshu MCP且已登录？
│   ├── 是 → 直接使用MCP（最高成功率）
│   └── 否 → 检查是否有完整xsec_token链接
│       ├── 有 → 用web_fetch（次选）
│       └── 无 → 提示用户扫码登录或提供带token链接
└── 如果只需要公开文字内容
    └── 可用web_fetch裸抓（最低质量）
```

### 详细执行步骤

#### 首选方案：xiaohongshu MCP
```bash
# 1. 检查MCP状态
ps aux | grep xiaohongshu-mcp
# 2. 若未运行，启动
./start-mcp.sh  # 会提示扫码登录
# 3. AI调用
调用 xiaohongshu.search(keyword="AI设计") 
或 xiaohongshu.get_note(url="https://...") 
# 4. 获取完整数据：正文、图片数组、点赞/收藏/评论数
```

#### 备选方案1：web_fetch + 完整链接
```bash
# 1. 检查链接是否含xsec_token
if "xsec_token=" not in url:
    # 提示用户提供带token链接，或跳过
    return "需要登录态，请提供完整链接或扫码登录"
# 2. 直接调用web_fetch
web_fetch(url)
# 3. 解析返回：仅文字内容，无统计数据
```

#### 备选方案2：curl SSR（仅技术验证用）
```bash
curl -H "User-Agent: Mozilla/5.0..." "$URL" | python3 parse_xhs.py
# 需要额外解析js变量，不推荐AI直接使用
```

### 失败处理
- **403/验证页面** → 立即切换到MCP方案或提示登录
- **空内容返回** → 检查链接完整性，要求xsec_token
- **频率限制** → 增加间隔至5秒以上，使用代理

### 最佳实践
1. **优先推广MCP**：一次登录，长期稳定
2. **用户教育**：告知用户如何获取带token的链接（浏览器登录后复制）
3. **降级预告**：明确告知web_fetch方案的局限性
4. **批量采集**：使用MCP的分页接口，避免单点频繁请求
