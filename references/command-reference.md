## 1. day / 每日记录

### 触发方式
- `day`
- `day 今天学了xxx`
- `day 只记关键事项`
- `每日`
- `日报`

### 执行流程
1. **读取上下文**
   - 当前对话内容
   - `~/.workbuddy/memory/MEMORY.md`
   - 今日已有的 `📅每日笔记/` 文件（避免重复）

2. **生成内容**
   - 使用模板 `templates/daily-note每日笔记.md`
   - 填充：今日所学、遇到问题、明日计划

3. **输出**
   - 文件：`📅每日笔记/YYYY-MM-DD.md`
   - 如文件已存在 → 追加到当日笔记

### Prompt 示例
```
用户：day
AI：读取今日已有笔记 + 对话上下文 → 生成/追加每日记录

用户：day 今天学了 React Hooks，遇到 useEffect 依赖问题
AI：以此为核心内容生成今日笔记

用户：day 只记关键事项
AI：生成简化版，只保留关键 bullet points
```

## 2. week / 周复盘

### 触发方式
- `week`
- `week 本周专注 AI 工具`
- `每周`
- `周报`

### 执行流程
1. **读取数据源**
   - 本周所有 `📅每日笔记/` 文件（按日期范围扫描）
   - 当前对话上下文
   - 全局记忆

2. **生成内容**
   - 汇总本周关键事项
   - 提取模式/洞察
   - 下周规划

3. **输出**
   - 文件：`📆每周笔记/YYYY-MM-DD-周复盘.md`

### Prompt 示例
```
用户：week
AI：扫描本周所有每日笔记 → 生成周复盘

用户：week 本周主要在做项目复盘
AI：以此为主题生成周复盘，侧重项目相关笔记
```

## 3. note / 主题收录（外部内容摘要）

### 触发方式
- `note https://example.com/article`
- `note 关于xxx的播客`
- `主题`
- `收录`

### 执行流程
1. **信息源路由**
   - 如有 URL → 查询路由表 → 调用对应 Skill 采集
   - 无 URL → 询问用户详情

2. **内容处理**
   - 提取核心观点（一段话总结）
   - 文章骨架 + 核心要点
   - 相关资源网络（同主题/同作者/核心概念/相关工具）

3. **自动子分类归档**（多篇同主题时才建立子分类）
   - 分析内容主题（AI工具/内容创作/开发实践/生活决策等）
   - 扫描 `📚主题笔记/` 下已有主题文件夹
   - 该主题已有 ≥2 篇笔记 → 归入对应主题文件夹
   - 该主题只有 1 篇或没有 → 直接放在 `📚主题笔记/` 根目录
   - 新主题达到 3 篇时 → 建议创建主题文件夹（如 `📚主题笔记/新主题/`）

4. **更新主题索引** ⭐ 关键步骤（借鉴 Karpathy Wiki）
   - 检查 `📚主题笔记/index.md` 是否存在
   - 不存在 → 创建根目录索引
   - 存在 → 按笔记 tags 分组追加
   - 格式：`- [[笔记名]] | 一句话摘要 | YYYY-MM-DD`
   - 结构：按 tags 多层分组，区分 note-overview / note-experience
   - 清理：「最近收录」只保留最近7天

5. **建立相关资源网络**
   - 扫描 `📚主题笔记/` 下已有笔记
   - 按四维度填入 `## 相关资源`：同主题、同作者、核心概念、相关工具
   - 核心概念提及 3 次+ → 建议创建概念卡片

6. **输出**
   - 文件：`📚主题笔记/<主题>/<topic-name>.md`
   - 使用模板：`templates/note-overview.md`
   - 索引：`📚主题笔记/index.md`（根目录统一索引，自动更新）

### Prompt 示例
```
用户：note https://www.example.com/ai-trends
AI：采集文章 → 判断分类为"AI工具" → 
    该主题已有3篇 → 归入 📚主题笔记/🤖AI工具/
    或该主题只有1篇 → 放在 📚主题笔记/根目录

用户：note 听了 Huberman 关于睡眠的播客
AI：询问要点 → 判断分类为"健康" → 归入对应位置
```

## 3b. exp / 经验沉淀（个人实践总结）

### 触发方式
- `exp 完成了xxx项目的复盘`
- `exp 搭建AI工作流的经验`
- `经验`
- `沉淀`

### 与 note 的核心区别

| 维度   | note（外部摘要）         | exp（经验沉淀）            |
| ---- | ------------------ | -------------------- |
| 内容来源 | 外部文章/视频/播客         | 个人实践/复盘/总结           |
| 模板重点 | 文章骨架 + 核心要点        | 实践骨架 + 关键决策          |
| 落点   | `📚主题笔记/<主题>/` | `📚主题笔记/<主题>/` |
| type | note-overview     | note-experience     |

### 执行流程
同 note，但使用 `templates/note-experience.md` 模板：
- **最简指南**替代**核心内容预览**
- 强调**实践骨架**（步骤/阶段）和**关键决策**
- 更适合记录个人方法论和踩坑经验

### Prompt 示例
```
用户：exp 完成了Obsidian+AI工作流搭建，总结一下经验
AI：提取实践步骤 → 识别关键决策点 → 
    归入 📚主题笔记/🤖AI工具/
    使用经验沉淀模板输出
```

## 4. idea / 创作素材来源

### 触发方式
- `idea 这篇文章的观点可以写成小红书`
- `idea 对标账号的选题角度不错`
- `灵感`
- `素材`

### 核心定位
**不是灵感收集或随想**，而是「创作的素材来源」——从别人的内容中标记出可以作为你输出来源的部分。

### 执行流程
1. **标记素材来源**
   - 记录来源内容的核心观点
   - 标注可输出的方向（写什么、发哪个平台）

2. **关联原始内容**
   - 如果有对应的 note 笔记，建立 `[[双向链接]]`
   - 如有原始 URL，保留 source 字段

3. **输出**
   - 文件：`📚主题笔记/创作灵感/<idea-name>.md`
   - 使用模板：`templates/note-overview.md`（启用「输出方向」章节）

### 与 note 的区别
| | note | idea |
|--|------|------|
| 目的 | 知识沉淀（学习用） | 输出素材（创作用） |
| 内容来源 | 外部文章/视频/播客 | 外部内容中可输出的部分 |
| 后续动作 | 存档、检索 | 用 `output` 展开为完整创作 |
| 落点 | `📚主题笔记/<子分类>/` | `📚主题笔记/创作灵感/` |

### Prompt 示例
```
用户：idea 这篇文章的 xxx 观点可以写成小红书
AI：记录素材来源 + 标注潜在输出方向（小红书）

用户：idea 对标账号的这个选题角度不错
AI：记录竞品选题灵感，标记为创作素材

用户：idea 从xxx视频提取的3个可输出点
AI：批量记录素材来源，每个标注输出方向
```

## 5. output / 创作输出

### 触发方式
- `output 写一篇关于xxx的文章`
- `output 把xxx主题的笔记整合成总览`
- `创作`
- `输出`

### 核心用途
**原创内容的最终落点**。可以是：
- 原创文章/帖子/分享稿
- 多篇笔记整合成的结构化总览（compile 场景）

### 执行流程

#### 场景 A：原创创作
1. 基于用户提供的主题/大纲
2. 可读取相关主题笔记作为参考
3. 生成原创内容
4. 输出到 `📝创作输出/`

#### 场景 B：多篇整合（原 compile）
1. 扫描 `📚主题笔记/` 中相关笔记
2. 提取每篇核心要点
3. 合并去重、理清逻辑
4. 生成结构化总览
5. 标注所有来源笔记的 `[[双向链接]]`

### 智能归档
分析内容主题 → 扫描 `📝创作输出/` 已有子分类 → 匹配归入或创建新类别

### Prompt 示例
```
用户：output 写一篇关于AI笔记工作流的文章
AI：原创创作 → 判断分类"AI工具" → 输出到 📝创作输出/AI工具/

用户：output 把AI工具主题的笔记整合成总览
AI：扫描相关笔记 → 合并生成结构化总览（compile 场景）→ 📝创作输出/AI工具/

用户：output 小红书风格，关于灵感管理的
AI：按平台调性创作 → 判断分类"小红书" → 📝创作输出/小红书/
```

## 6. todo / 任务跟踪

### 触发方式
- `todo 完成xxx功能`
- `todo 调研竞品`
- `任务`
- `待办`
- `task`（兼容旧指令）

### 执行流程
1. **解析任务**
   - 任务名称
   - 背景/上下文
   - 完成标准（必须具体可衡量）
   - 截止日期（如有）
   - 优先级（高/中/低）

2. **输出**
   - 文件：`📋待跟踪任务/<task-name>.md`
   - 使用模板：`templates/todo任务.md`

3. **后续更新**
   - 用户可随时说"更新任务 xxx"或"任务 xxx 进度"
   - AI 找到对应任务文件追加进度记录
   - 任务完成时更新 status: 🟢 已完成

### Prompt 示例
```
用户：todo 完成AI笔记Skill的README重写
AI：创建任务文件，包含背景、标准、进度追踪

用户：todo 调研3个竞品，周五前完成，高优先级
AI：创建带截止日期和优先级的任务

用户：task 完成xxx（兼容旧指令）
AI：同上，创建任务文件
```

4. **路由联动**

### Prompt 示例
```
用户：tool 小红书Skill可以采集笔记内容
```

## 8. classify / 收件箱整理

### 触发方式
- `classify`
- `classify 只整理最近3天的`
- `整理`
- `归类`

### 执行流程
1. **扫描收件箱**
   - 读取 `Clippings/` 中未分类文件

2. **逐篇判断**
   - 主题分类
   - 重要性评估
   - 是否值得转为正式笔记

3. **执行归档**
   - 生成精简版笔记到对应分类
   - 原始裁剪保留在 `Clippings/`（原始档案）

4. **输出摘要**
   - 分类结果统计
   - 建议后续动作

### Prompt 示例
```
用户：classify
AI：扫描所有未分类收件箱 → 逐篇判断 → 归档

用户：classify 只整理最近一周的
AI：限定时间范围整理
```

## 9. check / 健康检查（Lint 报告）

### 触发方式
- `check`
- `检查`
- `健康检查`

### 执行流程

#### A. 笔记库健康检查
1. **扫描全库**
   - 孤立笔记（无入链/出链）
   - 被多次提及但未建页的概念
   - 缺失 frontmatter
   - 过期内容（90天未更新）
   - 重复主题
   - 空收件箱
   - 缺失页面（被引用但不存在）

2. **输出 Lint 报告**（模仿 Karpathy 风格）

```markdown
## 📊 笔记库健康检查报告 | YYYY-MM-DD

### 🔴 孤立笔记（无入链）
| 笔记 | 位置 | 建议关联 |
|------|------|---------|
| [[某笔记]] | AI工具/ | → [[相关概念]] 或 → [[相似主题]] |
| [[另一笔记]] | 健康/ | 添加相关链接 |

### 🟠 高频概念待建页（借鉴 LLM-Wiki）
| 概念 | 提及次数 | 建议 |
|------|---------|------|
| "双引擎架构" | 5次 | 创建概念卡片 📚主题笔记/概念卡片/双引擎架构.md |
| "Zettelkasten" | 3次 | 创建概念解释页 |

### 🟡 过期内容（90天未更新）
| 笔记 | 最后更新 | 建议 |
|------|----------|------|
| [[旧笔记]] | 2026-01-15 | 复核内容是否仍有效 |

### ⚪ 缺失页面（被引用但不存在）
| 页面名 | 被引用于 | 建议 |
|--------|---------|------|
| "Obsidian" | 笔记A, 笔记B | 创建页面或移除链接 |

### 📈 知识网络统计
- 总笔记数：XX
- 孤立笔记：X (X%)
- 概念卡片：X
- 双向链接覆盖率：XX%
- 高频概念待建页：X个
- 最近7天新增：X篇

### 📋 建议操作（按优先级）
1. [高] 处理 X 篇孤立笔记
2. [高] 创建 X 个高频概念卡片
3. [中] 复核 X 篇过期内容
4. [低] 修复 X 个缺失页面
```

#### B. Skill 包一致性检查（增强）

**检查范围**：`obsidian-ai-notes skill配置/` 目录

**检查项**：
1. **文件完整性**
   - [ ] SKILL.md（核心执行规范）
   - [ ] README.md（英文用户文档）
   - [ ] README_CN.md（中文用户文档）
   - [ ] templates/（所有模板文件）
   - [ ] references/command-reference.md（指令参考）

2. **内容一致性**
   - [ ] 指令表一致（SKILL.md ↔ README.md ↔ command-reference.md）
   - [ ] 文件夹结构描述一致
   - [ ] 模板文件名与指令引用一致
   - [ ] 版本号一致

3. **引用正确性**
   - [ ] 模板文件存在且可被引用
   - [ ] 文档间交叉链接有效
   - [ ] 指令别名完整（如 task/todo）

**输出格式**：
```markdown
## 🔧 Skill 包一致性检查

### ✅ 文件完整性
- [x] SKILL.md
- [x] README.md
- [x] README_CN.md
- [x] templates/note-overview.md
- [x] templates/note-experience.md
- [x] templates/todo任务.md
- [x] references/command-reference.md

### ✅ 内容一致性
| 检查项 | SKILL.md | README.md | command-reference.md | 状态 |
|--------|----------|-----------|---------------------|------|
| 指令数量 | 10 | 10 | 10 | ✅ |
| note 落点 | note-overview摘要/ | note-overview摘要/ | note-overview摘要/ | ✅ |
| exp 指令 | ✅ | ✅ | ✅ | ✅ |
| todo 指令 | ✅ | ✅ | ✅ | ✅ |

### ⚠️ 发现的问题
- [问题描述及修复建议]

### 📋 建议操作
1. [高] 修复 XXX 不一致
2. [中] 更新 XXX 描述
```

3. **可选修复**
   - 用户确认后，AI 可自动执行修复

### Prompt 示例
```
用户：check
AI：全库扫描 + Skill包一致性检查 → 输出完整报告

用户：check 只看主题笔记
AI：限定范围检查 → 输出该分类报告

用户：check 并修复
AI：扫描 → 输出报告 → 询问修复确认 → 执行修复

用户：check --skill-only
AI：只检查 Skill 包一致性
```
